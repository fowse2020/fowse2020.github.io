在GitHub上搭博客之Hexo篇

这是一个极其初级的新手教程，由于本人水平有限，纰漏之处还望见谅。不要怕代码、动手DIY，多Google一下就好了。

2019年1月28日，更新了Hexo升级操作；2018年10月27日更新了章节顺序并微调了操作内容。

共分为5个步骤，分别是：

  1. 安装Node.js和Git
  2. 注册一个后缀为.io的GitHub项目
  3. 生成SSH密钥并添加到GitHub
  4. 安装并配置Hexo通过Git方式更新
  5. 撰写和发布文章
  6. 更新Node.JS及Hexo程序

1. 安装Node.js和Git

安装这两个工具的步骤不再详述，请到各自官网查阅或搜索安装教程即可。其中Node.js这个版本帝推荐安装LTS(Long Term Support)版本。Windows环境可能要设置一下环境变量并重启系统、Mac和Linux则相对简洁。

Windows下可使用choco命令行工具（详见我的另一篇文章），Mac下可用brew命令安装，Debian系的发行版用apt-get、RedHat系发行版用yum安装即可，安装过程中不可避免会遇到问题，请自行搜索解决。

安装成功后在命令行输入node -v和npm -v查看版本号及是否安装成功。Git和Node.js自带的npm是后续步骤的前提，请先确保正确安装。

2. 注册GitHub并创建一个XXX.github.io项目

首先你要在GitHub上注册一个帐号，设定好你的用户名和注册邮箱（这两项内容下面会用到，很重要）。

登录后点击“New repository”，根据自己的喜好给项目起个格式为“xxx.github.io”的名字，选择公开项目，点击创建；同时在Pages中开启Github Pages服务。

等搭建完成，你的项目名将作为你的博客地址，形式如：myblog.github.io

在GitHub上添加SSH Key（公钥）

首先，登录你的GitHub，点击右上角的个人头像，进入设置选项。

在设置页面的左侧，点选SSH and GPG Keys，在打开的页面中，点击右上角的New SSH Key，如下图所示。

页面进入Add new之后，首先在下图1所示的Title框给你的密钥取个名字，例如我的电脑是ThinkPad x230这样命名就知道是哪个电脑在更新博客了。然后将“创建密钥”步骤中打开的公钥复制(以ssh-rsa开头的一长串字符)，粘贴到下图2的Key文本框中，点击下图3所示的添加按钮。

随着GitHub的更新，有教程说需要添加的SSH Key是项目设置里的Deploy keys。即在项目设置界面，点选左侧的Deploy keys，选择Add deploy key，点击Add deploy key。然后把创建的id_rsa.pub里的内容复制到Key里去，Title部分自定义一下，点击Add key保存即可。读者可自行测试。

至此，你就可以通过执行Hexo的简单命令上传和更新你的博客了。

建议使用Hexo的官网给出的安装教程，在网络状况良好的情况下，用户只需要几行命令和稍加等待就可以顺利安装Hexo程序。

5. 撰写并发布你的第一篇文章

通过命令创建文章，通过Markdown编辑器撰写文章，通过Hexo（bash）命令生成和发布博客。

  • hexo clean # 清除缓存命令，当你修改了任意本地文件之后，建议先执行这个命令，再执行下面的g和d两个命令
  • hexo g # hexo generate命令的缩写，在本地生成静态html文件，你可以通过hexo s命令本地预览博客内容
  • hexo d # hexo deploy命令的缩写，一键发布本地内容到GitHub上

执行最后一个命令后，会显示INFO Deploy done: git的提示，这表示发布成功了。当然你可以对本地文件反复修改，直到自己满意，然后运行上述命令更新之后，就可以在“xxx.github.io”看到自己刚才撰写和修改后的博客内容了。如果发布失败，请大家根据错误提示，搜索相关教程。
